JournalTrace

JournalTrace – программа, которая позволяет обнаружить любые действия на компьютере: перемещение, удаление или переименование файлов/папок.

Как начать пользоваться

1 Нажмите "Drive" > "Select" > C:\ > "Select"
2 Нажмите опять "Drive" > "Scan"
3 Дождитесь зеленой полоски и сообщения "All done!"
4 Нажмите "Layout" и выберите "Data Grid" или "Directory Tree"

Типы сортировки

Data Grid – помогает находить файлы по имени или папке. Есть фильтры: Directory, Name.

Directory – ищет исключительно по директориям. Вставляем под этот фильтр "Адрес папки", пример: C:\Users\System32\Desktop\ и выбираем "Directory". По фильтру "Name" Вы там ничего не найдёте.
Directory Tree – подходит для просмотра папок, которые расположены возле C:\ > $Recycle.Bin, downloads, prefetch.

Object ID — это уникальный номер (16-байтный GUID) файла или папки в Windows. Он не меняется, даже если файл преименовали или перенесли в другое место.

Как найти очистку USN-Journal

1 Откройте "Data Grid"
2 Сортируем USN, кликая по нему два раза
3 Если число 0 – может быть очистка
4 Заходим Everything и ищем файл по фильтру: ext:pf fsutil
5 Если .pf за сегодняшнее число – блокируем за очистку
6 Если нету – применяем EventVwr

System Informer

Для корректной работы рекомендуется использовать Canary-версию System Informer.

Настройка

1 Запустите System Informer от имени администратора
2 Перейдите в Options → Enable Kernel-Mode Driver
3 Если появится сообщение об ошибке, нажмите OK и продолжайте

Анализ строк процесса

1 Дважды щелкните по процессу
2 Перейдите на вкладку Memory
3 Выберите Options → Strings
4 Установите Minimum Length: 4
5 Включите: Extended Unicode, Mapped
⚠ РЕКОМЕНДУЕТСЯ произвести reset настроек во избежание утечки методов проверки/фикса строк на читы. Для сброса: System → Options → Reset

WinPrefetchView

Все данные считываются из Prefetch-файлов Windows. Расположение: C:\Windows\Prefetch\. Расширение файлов: .pf

Какие данные можно получить?

  • Имя исполняемого процесса
  • Полный путь к .exe
  • Количество запусков программы
  • Дату и время последних запусков
  • Список DLL и файлов, загруженных во время запуска
  • Информацию о диске (томе)
  • Версию Prefetch-файла

Ключевые процессы для анализа

ПроцессОписание
cmd.exeОпределяет использование командной строки (.bat, .cmd файлы)
conhost.exeПодтверждает использование командной строки, факт запуска консольных программ
consent.exeUAC - запуск программ с правами администратора
powershell.exeАвтоматические скрипты, загрузчики, утилиты
explorer.exeПроводник Windows
regedit.exeРедактор реестра
msiexec.exeУстановка программ
rundll32.exeЗапуск DLL
wscript.exe / cscript.exeВыполнение VBScript и JScript
java.exeКонсольная версия Java
⚠ Windows создаёт Prefetch не для каждого процесса. Очистка папки Prefetch может удалить часть информации.
Наличие Prefetch-файла само по себе не доказывает использование запрещённого ПО. Он лишь подтверждает факт запуска соответствующего исполняемого файла и должен оцениваться в совокупности с другими артефактами.

Shellbags

Shellbags – программа, которая опирается на ветку Shell/Bags в реестре. Позволяет найти следы папок, которые Вы открыли, и когда Вы его открыли.

Как найти очистку

В программе Shellbags Analyzer & Cleaner можно очистить shell/bags ветки в реестре. Чтобы найти очистку, откройте Everything напишите: shellbag. Смотрим файлы формата ".ini" – они создаются при очистке "клинером".

Как начать пользоваться

1 Нажмите снизу кнопку "Анализ"
2 После анализа нажимаем сверху слева "Показать всё"
3 Удаленные папки – показывает за все время удаленные папки
4 Сортировка: два раза нажимаем на "Посещенные"
5 Существующие папки – можно глянуть папки, которые есть на пк

EventLog

Event Log – встроенная программа Windows. Она делает записи информации о важных действиях, ошибках, предупреждениях.

Как открыть

Нажмите Win+R, напишите eventvwr, Enter.

Важные ID событий

IDОписаниеГде искать
4616Редактированное времяЖурнал Windows > Безопасность
7045Драйвера читовЖурнал Windows > Система
1102Очистка EventVwr (второй файл)Журналы Windows > Безопасность
104Очистка EventVwrЖурналы Windows > Система
3079Очистка USN-JournalЖурналы Windows > Приложение

Как обнаружить редактирование времени (4616)

Когда пользователь меняет время, он может чистить journal, eventlog, выключать службы. Находится в: Журнал Windows > Безопасность > ID 4616

Драйвера читов (7045)

Может быть такое что чит использует драйвер для очистки. Находится: Журнал Windows > Система > ID 7045

Очистка EventVwr (104)

Пользователь может очистить сам ивент. Код 104 в "Журналы Windows" > "Система" является очисткой eventvwr.

Очистка USN-Journal (3079)

Опытный пользователь может удалить USN-Journal командой: fsutil usn deletejournal /D C:

Заметить можно в JournalTrace: если число USN = 0, значит пользователь очистил usn.

Для подтверждения: EventLog > Журналы Windows > Приложение > код 3079

Редактированное время PowerShell (4100)

В PowerShell Operational можно увидеть предупреждения (Warning) с контекстом выполнения команд.

Cdpusersvc

Отвечает за файлы типа Activitiescache.db, позволяет передать информацию в такой файл для фильтрации.

Шаги

1 Запустите SystemInformer
2 Сделайте дамп службы cdpusersvc
3 Фильтруем:
{"displayText" --> .exe ,"activationUri" platform --> .exe x_exe_path --> .com @{"displayText"

"-->" — значит, после определенного действия. Пишем сначала: {"displayText", а потом уже .exe

Power / PlugPlay / DCOMLaunch

Power – процесс, который обрабатывает запросы и настраивает энергопотребление компьютера. От него питаются службы dcomlaunch и plugplay. Через него ищем .jar файлы.

Настройка

Те же самые галочки что и для других процессов, но ставим "5".

Фильтры

Поиск .jar файлов:
(Contains case-insensitive) > .jar
Поиск переименованных файлов:
(Contains case-insensitive) > -jar
⚠ Данный процесс нестабильный, часто "забывает", так что большая вероятность, что там ничего не будет, хотя пользователь будет использовать софт.

csrss

csrss – критически важный системный процесс Windows. Отвечает за работу пользовательских графических элементов.

Процесс csrss с меньшим размером байтов (EXE)

Regex (case-insensitive):
^[A-Z]:\\.+\.(exe)$

Процесс csrss с большим размером байтов (DLL)

Regex (case-insensitive):
^[A-Z]:\\.+\.(dll)$

Другой регекс

Regex (case-insensitive):
((?!exe|pyd|manifest|dll|config|\\|cpl|microsoft-|shell).)*$

Для обнаружения dll и exe одновременно

Фильтр:
^[a-z]:.+\. — подойдет для PathParser

MFTECmd

NTFS хранит информацию о действиях с файлами: время создания, изменения, последнего доступа и записи. Также могут сохраняться дополнительные потоки данных (ADS), например "Zone.Identifier".

Как использовать

1 Запустите командную строку от имени администратора
2 Обозначьте папку: cd путь
3 Впишите: MFTECmd.exe -f C:\$MFT --csv .
4 Откройте полученный CSV в Timeline Explorer

Что анализировать

  • Directory — расположение файла
  • Last Access (0x30) — время последнего доступа
  • Zone ID Contents — источник файла (URL)

AnyDesk

Продвинутый пользователь может заддодисить интернет через AnyDesk.

Как защититься

Нужно отключить в настройках Anydesk "прямое подключение":

Настройки > Соединение > Прямое подключение — отключаем галочку (стоит по умолчанию включённой).

AnyDesk bypass (anynet_unknown)

Пользователь может забайпасить бесконечным подключением. На экране будет: "anynet_unknown".

⚠ За этот байпас мы не можем забанить сразу. Нужно убедиться, что пользователь пытается скрыть экран. Если ошибка >3 раз подряд — выдаём блокировку.

Как повторить ошибку

Первый метод (лёгкий): Закрываем Anydesk в трее, когда появилось окно подключения.

Второй метод (эффективный): Когда появилось окно о подключении, открываем вкладку и много раз нажимаем "Переподключение".
